h5py Performance Benchmarks

Time-based Benchmarks (Lower is Better)

These benchmarks measure execution times for various h5py operations. Lower values indicate better performance.

  • Write Time: Time to write data to HDF5 file
  • Sequential Read Time: Time to read data sequentially
  • Threaded Read Time: Time to read data using multiple threads

Speed-based Benchmarks (Higher is Better)

These benchmarks measure throughput speeds for various h5py operations. Higher values indicate better performance.

  • Data Saving Speed: Uncompressed data writing throughput
  • Effective Write Speed: Actual file writing throughput
  • Sequential Read Speed: Sequential data reading throughput
  • Threaded Read Speed: Multi-threaded data reading throughput

About These Benchmarks

These benchmarks are automatically run daily against the latest HDF5 development version from GitHub. The benchmarks use the slicing benchmark from h5py's test suite, which exercises pathological read patterns across chunked datasets to stress-test I/O performance.

Test Environment:

  • Ubuntu Latest (GitHub Actions)
  • Latest HDF5 from develop branch
  • Python 3.11
  • Dataset: 1024³ float32 volume with 64³ chunks
  • Test pattern: Orthogonal hyperplane reads (worst-case scenario)

Results are stored using github-action-benchmark and updated automatically with each benchmark run.
